#Definition of Done
A story is considered DONE only when ALL the following are true:
Working & Verified
• Acceptance criteria manually checked
• At least one failure case handled
• No crashes - errors return meaningful responses
Code Quality Baseline
• Naming and formatting follow class standard.
• Route is thin (business logic not embedded in routing).
• No unsafe !! without justification.
• No obvious duplicated logic.
Testing
• At least one of the following:
• One automated test, OR
• A written manual test checklist that was run.
Pull Request & Integration
• Opened as a PR (not pushed directly to main).
• At least one teammate reviewed it.
• Reviewer confirmed:
• Code is understandable.
• It meets the DoD.
• It doesn’t break existing functionality.
• PR builds and runs after merge.
